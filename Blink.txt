PROGRAMME ANALYSEUR D'AIR (CO2, Température, Humidité)
======================================================
Ce programme transforme un ESP32 C3 en une station de monitoring environnemental avancée avec écran TFT.

RÉSUMÉ DES FONCTIONNALITÉS :
- Connexion WiFi automatique avec synchronisation de l'heure via NTP.
- Lecture du capteur CO2 SCD40 via le bus I2C.
- Affichage temps réel sur écran TFT ST7789 sans clignotement.
- Mode Nuit intelligent (22h - 07h) : extinction COMPLÈTE de l'écran (rétroéclairage et contenu).
- Suivi des records de journée (07h - 22h) et Records de nuit (22h - 07h).
- Cycle d'affichage intelligent : alterne toutes les 5s entre Temps Réel, Records Nuit et Records Jour, UNIQUEMENT si des données ont été enregistrées pour ces périodes.
- Interface réactive (non-bloquante) avec curseur de statut clignotant.
- Optimisation de consommation avec un rétroéclairage PWM et des délais système intelligents.

DESCRIPTION DES FONCTIONS DU CODE :
-----------------------------------

1. setup()
   - Initialise la communication Série, le bus SPI pour l'écran et le bus I2C pour le capteur.
   - Configure le rétroéclairage de l'écran via PWM (ledc).
   - Établit la connexion WiFi et configure l'heure réseau (NTP).
   - Démarre la mesure périodique du capteur SCD40.
   - Initialise le décompte de démarrage (1 minute).

2. loop()
   - Fonction principale non-bloquante.
   - Gère le timing de lecture du capteur (1000ms) et du clignotement du curseur (500ms).
   - Incorpore un délai de 10ms pour la gestion CPU.

3. processStateAndDisplay(co2, temp, hum)
   - Logique métier principale et gestion des états temporels.
   - Suivi des données : 
     * Enregistre les Extremes Nuit (Max CO2, Min Temp, Max Hum) de 22h à 07h.
     * Enregistre les Extremes Jour (Max CO2, Min Temp, Max Hum) de 07h à 22h.
   - Gestion de l'affichage :
     * De 22h à 07h : Éteint le rétroéclairage et vide l'écran.
     * De 07h à 22h : Alterne cycliquement entre les modes disponibles (Temps Réel + Records si disponibles) toutes les 5 secondes.
   - Gestion des Resets : Réinitialise proprement les records au début de chaque cycle (07h pour le jour, 22h pour la nuit).

4. updateBacklight(bool on)
   - Contrôle le transistor de rétroéclairage via PWM.

5. displayUI(co2, t, h, label)
   - Moteur graphique optimisé (sans scintillement).
   - Ne rafraîchit l'écran complet que lors d'un changement de libellé de mode.
   - Met à jour les valeurs uniquement si elles changent, par surimpression (TextColor avec fond).
   - Utilise un formatage fixe pour éviter les artefacts de décalage.
